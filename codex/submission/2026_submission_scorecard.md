# Submission Scorecard (Aligned to Gemini 3 Hackathon Rules)

Score each 1-5 before final submit.

## Weighted criteria
- Technical Execution (40%): ____ / 5
- Innovation/Wow Factor (30%): ____ / 5
- Potential Impact (20%): ____ / 5
- Presentation/Demo (10%): ____ / 5

Weighted total =
`0.40*T + 0.30*I + 0.20*P + 0.10*D`

## Hard-gate checklist (Stage One)
- [ ] Public link to working project/demo.
- [ ] Public code repo (or valid AI Studio app link path).
- [ ] Demo video <= 3 minutes, English (or subtitles).
- [ ] Submission text includes Gemini integration details.
- [ ] Project clearly addresses a challenge and uses Gemini 3 centrally.

## Technical Execution checks
- [ ] Primary user flow succeeds live in one take.
- [ ] Error states are handled and visible.
- [ ] Structured outputs or deterministic parsing used where needed.
- [ ] No dead buttons / placeholder paths in demo.

## Innovation checks
- [ ] Signature feature is differentiated from standard chatbot pattern.
- [ ] Interaction model itself feels novel.
- [ ] Judges can explain novelty in one sentence after watching.

## Impact checks
- [ ] Target user is specific.
- [ ] Pain point is real and immediate.
- [ ] Expected measurable benefit is stated.

## Presentation checks
- [ ] Problem and value are clear in first 15 seconds.
- [ ] Real app usage dominates video time.
- [ ] Architecture/technical explanation is concise and clear.
- [ ] Claims match what is shown on-screen.
