# 4-Minute Demo Blueprint (Live-First, Challenge-Centric)

Date: 2026-03-02  
Priority rule: Live feedback + challenge loop first. DNA/pose/chat controls are secondary.

## Demo timing (4:00 total)

### 0:00-0:20 — Hook
- One-line problem: solo bowler has no instant expert feedback.
- One-line solution: Gemini Live mate + challenge pressure loop.
- Show app already on camera view, ready to bowl.

### 0:20-3:00 — Primary proof (Gemini Live + Challenge)
- Start live session.
- Bowl delivery; on-device detection + spoken count confirms event.
- Ask mate verbally; Gemini Live responds with concise spoken feedback.
- Activate challenge target ("Try yorker on off stump").
- Bowl again; evaluate hit/miss and update challenge score aloud + on screen.
- Repeat once to prove loop consistency (target -> bowl -> evaluate -> score).
- Briefly show reconnect/fallback behavior without breaking flow.

### 3:00-3:50 — Secondary add-ons (only if stable)
- 5s clip auto-analysis card (pace band + one insight).
- Quick DNA similarity glimpse.
- Quick phase chat/jump control demonstration.

### 3:50-4:00 — Close
- State measurable value (engagement + feedback speed + challenge adherence).
- Clear boundary: what is shipped now vs what is next.

## Hard constraints
1. Live segment must dominate (minimum 3 minutes).
2. No speculative claims that are not shown in-video.
3. If any secondary feature is unstable, skip it live and keep focus on primary loop.
4. First magic moment must occur before 0:45.

## Judge alignment
- Technical Execution (40%): uninterrupted primary loop + visible fallback.
- Innovation/Wow (30%): spoken challenge loop, not generic chat.
- Impact (20%): clear user and measurable training value.
- Presentation (10%): fast hook, product-first footage, explicit Gemini role.
